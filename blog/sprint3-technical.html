The differences between HTML and CSS
Do you like cats? I do. Let's take my Kotya cat as an example to describe the difference between HTML and CSS.
Think of HTML as my cat's skeleton. Yes, HTML is like the skeleton of the page. With HTML you specify the structure of your page by defining what elements it will have and semantics of the elements (header, footer, navigation).
On the other hand, CSS is like my cat's exterior - well-built body, fluffy hair, ginger eyes. How about a cute small hat on the head? So, CSS provides the color, sizing, scaling, and other subtle touches that will make your web page more appealing to look at.
Control flow and loops in our life
Control flow is the order that the instructions or statements or functions are executed. Code is run in order from the first line in the file to the last line, unless we have conditionals and loops that change the control flow.
If I think about control flow and loops in my life, I can find many examples. One of them is making russian pancakes (blini). I make pancakes every weekend for my family. The "Pancake Code" looks like:
  1. In a large mixing bowl, whisk eggs with oil then add milk.
  2. Add sugar, salt, baking soda. Mix all ingredients.
  3. Add flour in equal parts to the milk and egg mixture; mixing well until no lumps are visible.
  4. Heat a medium non-stick pan over medium heat.
  5. Pour the batter into the center of the pan and swirl to spread evenly.
  6. Flip the pancake over, if the surface looks porous and sides are golden.
  7. Remove the pancake to a serving platter.
  8. Repeat 5, 6 and 7 until all batter is gone.
As you can see we can find in this "code" if/else statement (check if pancake is golden colour before you flip it/remove to a plate) and loop (continue until all batter is gone).
What is the Document Object Model?
The Document Object Model (DOM) is a programming API for HTML. The Document Object Model represents each of these web pages in a tree-like structure to make it easier to access and manage the elements. With the Document Object Model, programmers can build documents, navigate their structure, and add, modify, or delete elements and content.
The DOM represents the document as nodes and objects. That way, programming languages can connect to the page. Nodes can also have event handlers attached to them. Once an event is triggered, the event handlers get executed.
Let's look at an example - Facebook messenger. You open it, type a message and push the "Send" button or press the "Enter" key. That message is added into your messenger and it will stay in your messenger history.
the difference between accessing data from arrays and objects
An array is a special value, which is used to store multiple values at a time.
Example:
var animals = ["cat", "dog", "hamster"];
We can access an array element by adding an index in square brackets to the variable name. The following code displays "cat":
console.log(animals[0]);
An object is a value too. It's like a collection of key - value pairs. You can use “dot notation” to access the value by the key. It looks like this:
var cup = {count: 3, color: "green", material: "ceramic"};
console.log(cup.color);                // "green"
Properties of JavaScript objects can also be accessed using a bracket notation. If property name has a space or property is a dynamic variable, it can only be accessed using the square bracket notation. 
var cup = {count: 3, color: "green", material: "ceramic"};
console.log(cup["material"]);          // "ceramic"
Functions in JavaScript
Functions are the main “building blocks” of the program. They allow the code to be executed without repetition as many times as you want. Also, functions make a code simple and Functions allow a programmer to divide a big program into a number of small and manageable functions.
