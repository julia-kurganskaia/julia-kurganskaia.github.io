Classes vs IDs: Best Practices
A working knowledge of HTML and CSS lets you do whatever you want with your web site. For this reason it's important to know when and how to use HTML classes and IDs.
Classes and IDs are hooks you can assign to HTML elements to make them more specific. That you have additional identifiers to assign CSS markup to. So, when to use which? Classes and IDs basically do the same thing. They are used to apply styles to elements based on the class and ID assigned to an HTML element. However, there are differences between them.
ID's are unique
  • Each element can have only one ID
  • Each page can have only one element with that ID
Classes are not unique
  • Class can be used to identify more than one element
  • You can use multiple classes on the same element
As you can see ID is used for single elements that appear on the page for only once (e.g. header, footer, menu), whereas class is used for single or multiple elements that appear on the page for once or more than once (e.g. paragraphs, links, buttons). Element can have both an ID and a class. In most cases, it is preferable to add a class to an element instead of an ID. If you are not sure wich one to use, you should ask yourself: "Is there more than one element which requires the same style, now or at any time in the future?", and the answer is even "maybe", then make it a class.
